# Sample datasets

Altera AI ships with sample datasets used for:

- Unit tests (small, deterministic fixtures).
- Local development (a realistic-but-small mock dataset per
  methodology).
- Demos (a few hundred rows, sanitised, no real retailer data).

Datasets live under `tests/fixtures/` (small) and
`apps/api/altera_api/sample_data/` (larger).

## Per-methodology fixtures

Because the two methodologies have different required columns and
different headline outputs, fixtures are organised per methodology.

## Required and optional columns by methodology

See [input-formats.md](input-formats.md) for the canonical column
list per methodology. Cross-methodology fixtures supply the union of
both column sets.

Disallowed at ingestion (dropped silently with an audit entry):
`sales_value`, `revenue`, `margin`, `cost_price`, `supplier_*`,
`store_*`, `promotion_*`, `confidential_*`, `internal_*`. See
[../classification/ai-inputs-policy.md](../classification/ai-inputs-policy.md).

## Fixture conventions

- All fixtures are committed in the repo. None of it is real retailer
  data.
- Brand names in fixtures are fictional. Where a real-sounding brand
  appears, it is annotated as `[fixture brand]` in a comment in the
  generating notebook.
- Fixtures intended for unit tests include a sibling
  `*.expected.json` file with the expected classification and
  calculation outputs.

## Generating fixtures

Fixtures are generated by scripts under `scripts/fixtures/`. They are
not manually maintained CSVs. The scripts are deterministic and
seedable; running them produces byte-identical output across machines
for a given seed.

## Worked example — WWF composite

`wwf_step2_ingredients.csv` contains a 400g own-brand vegan lasagna
(`is_own_brand=true`, `wwf_is_composite=true`,
`wwf_composite_step1_bucket=vegan`). The accompanying JSON supplies:

```json
{
  "P-VL-001": {
    "ingredients": [
      {"food_group": "FG1", "subgroup": "alternative_protein_sources", "ingredient_weight_kg_per_item": 0.070},
      {"food_group": "FG2", "subgroup": "dairy_alternative_plant", "ingredient_weight_kg_per_item": 0.020},
      {"food_group": "FG5", "subgroup": "refined_grain", "ingredient_weight_kg_per_item": 0.100},
      {"food_group": "FG4", "subgroup": null, "ingredient_weight_kg_per_item": 0.150}
    ]
  }
}
```

The fixture's `expected.json` verifies that:

- Step 1: the full 400g per item is recorded against `vegan` and
  `total_composite_weight_kg`.
- Step 2: 70g per item is attributed to FG1 plant-alternative, 20g per
  item to FG2 plant-alternative, 100g per item to FG5 refined grain,
  150g per item to FG4 — and the 60g per item remainder is unreported
  residual.
