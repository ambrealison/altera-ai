# Testing strategy

Altera AI's tests are organised around the parts of the system whose
correctness has direct customer impact. Other code is tested as needed,
but the following areas are **required** to have automated tests:

- Input validation.
- Unit conversion and normalisation.
- Deterministic classification (rules engine).
- Protein Tracker calculations.
- WWF calculations.
- Manual review state transitions.
- AI JSON parsing (with fakes; the real provider is not called in CI).
- RLS policies (against a real Postgres instance).
- AI inputs policy enforcement (no commercial fields leave the
  process).

## Test taxonomy

| Layer        | Tooling     | Scope                                           |
|--------------|-------------|-------------------------------------------------|
| Unit         | pytest      | Pure-function behaviour. No I/O.                 |
| Integration  | pytest      | Validation, rules engine, manual review state,  classification orchestration with fake providers. |
| Database     | pytest + Supabase local | RLS policies, migrations apply cleanly, seed data is consistent. |
| API          | pytest + httpx | FastAPI routes against a test database.          |
| Frontend     | Vitest + React Testing Library | Component behaviour, accessibility.    |
| End-to-end   | Playwright  | A small set of golden-path flows: upload → classify → review → run → export. |
| Reproducibility | pytest    | Tiny fixture inputs produce byte-identical expected outputs. |

## Fixtures

Fixtures live alongside the code they test where they are local in
scope; cross-cutting fixtures (the `tiny_*` CSVs) live under
`tests/fixtures/`. Fixtures are generated by deterministic scripts; see
[docs/data/sample-datasets.md](../data/sample-datasets.md).

## Property tests

For unit conversion and the calculation modules, we use Hypothesis to
generate input grids and assert invariants:

- Output is bounded by `[0, total]` per row.
- For PT rows that use the per-product composite split extension,
  plant + animal protein equals total protein within `Decimal`
  precision.
- The PT 50/50 composite default is exactly 50/50 within `Decimal`
  precision when applied to the residual composite total.
- For WWF, FG2 dairy-equivalent factors are exactly ×10 for cheese
  and ×1 for other dairy.

## CI

The CI workflow runs, in order:

1. Lint + format check (Ruff, Prettier).
2. Type check (mypy, tsc).
3. Backend unit + integration tests.
4. Database migration + RLS tests.
5. Frontend unit tests.
6. Reproducibility tests (the byte-identical fixture check).
7. End-to-end tests (a smaller golden-path set).

A failing reproducibility test blocks merge unless the PR also
includes a methodology, taxonomy, or rules version bump and an updated
`*.expected.json`.

## What we do not test

- Third-party library internals.
- Hot-path log lines.
- Speculative scenarios that the product does not actually expose.
